## Policy

- In-app interface symbols should use SF Symbols where permitted by Apple's platform license and should not be redistributed as standalone artwork.
- The app icon and release artwork must be original project work or use assets with a redistribution-compatible license.
- Record the asset path, creator/source, date, license, modifications, and any attribution requirement in this file before committing the asset.
- Do not use third-party logos, device artwork, screenshots containing personal data, or generated imagery with unclear redistribution rights.
- Keep release-asset source hashes, transformation boundaries, and verification results in `RELEASE-ASSET-PROVENANCE.md`; never add a planned asset to this ledger before the file exists and has been inspected.

Original project artwork will be distributed under the repository's MIT License unless a per-asset entry states otherwise.
