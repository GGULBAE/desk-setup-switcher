# UI refinement goal

현재 `Desk Setup Switcher`의 안전한 적용·롤백 구조를 유지하면서 메뉴 막대와 Settings UI를 사용자가 빠르게 이해하고 실수 없이 조작할 수 있는 수준으로 개선하라.

단순한 시각 목업, 문구 제안 또는 계획에서 멈추지 말고 실제 SwiftUI 구현, 결정론적 테스트, 영문·한국어 현지화, 읽기 전용 화면 검토, 문서와 CI 증거까지 완성하라. 안전한 범위에서는 자율적으로 계속 진행하되 실제 시스템 설정 변경, 권한 부여, 개인정보가 포함된 증거 저장 또는 제품 방향을 크게 바꾸는 결정이 필요한 경우에만 사용자 확인을 요청하라.

## 상태와 기준선

- 상태: complete
- 구현 기준선: `9cd391f0acdec828aecbe5d202458cc86808f22b`
- 기준 화면: 2026-07-12에 설치된 한국어 앱의 메뉴 팝오버, 적용 미리보기, 프로필, 권한, 앱 정보 화면을 읽기 전용으로 검토했다.
- 화면 검토 중 실제 설정 적용, 강제 적용, 현재 설정 캡처, 위치 요청, 로그인 항목 변경 또는 진단정보 열람은 수행하지 않았다.
- 기준 스크린샷에는 실제 장치 식별정보가 포함될 수 있어 저장소에 추가하지 않는다. 이 문서의 관찰과 완료 기준을 영구 기준으로 사용한다.
- 현재 구현: 앱 생명주기 초안 보호, 값 중심 요약·미리보기, 메뉴 행동 계층, 조건 선택·검증, 권한/About/진단/접근성 메타데이터, 영문·한국어 구조 검사가 소스에 반영되고 전체 `make verify`가 통과했다.
- 현재 자동 증거: 기본 비라이브 스위트 214개가 통과했다. XCTest 111개 중 5개와 Swift Testing 103개 중 1개는 명시적 옵트인 테스트라 건너뛰었고, 프레젠테이션 전용 55개는 초안 XCTest 28개와 표시·조건 Swift Testing 27개다.
- 현재 로컬 증거: 완료 후 메뉴 단순화까지 포함한 트리에서 universal Debug/Release, Analyze, DMG·체크섬·마운트·리소스·아키텍처·ad-hoc 서명 분류가 통과했고 로컬 DMG SHA-256은 `a6c539267b1103537d041c6181ee822356db69c91ecf8e6467ccd8c7154d6473`이다.
- 현재 구현 증거: UI-hardening 커밋 `5f0cabc`가 `origin/master`에 push되었고 [GitHub Actions `29181900967`](https://github.com/GGULBAE/desk-setup-switcher/actions/runs/29181900967)에서 전체 `make verify`와 unsigned artifact ID `8256718472` 업로드가 성공했다. 다운로드한 CI DMG SHA-256은 `f3d82b033e8e375c9063a9b72cbd174d94a03f0cdd4414961895db3b3dcfc3f4`로 체크섬 파일과 일치했다.
- 현재 증거 경계: 현재 트리 스크린샷, 전체 키보드·VoiceOver, TCC, 실제 하드웨어 읽기·변경, Keychain write는 수행하지 않았다.

### 완료 후 메뉴 단순화

2026-07-12 후속 사용자 요청으로 프로필별 `가용성 검토`와 메뉴의 수동 `준비 상태 새로 고침` 컨트롤을 제거했다. 준비 상태는 메뉴를 열 때 계속 자동으로 읽는다. 각 프로필에는 해당 초안을 바로 선택해 Profiles 탭을 여는 `프로필 수정` 행동을 추가했고, 일반 설정 행동도 accessory 앱 활성화 뒤 공개 `OpenSettingsAction`을 호출하도록 교체했다. 다른 dirty 초안이 있으면 기존 저장·버리기·취소 보호를 그대로 사용한다. 내부 review-only 계획 모델과 결정론적 테스트는 회귀 검증을 위해 유지하지만 메뉴에서는 노출하지 않는다. 이 후속 트리는 live 플래그 없이 전체 로컬 `make verify`를 통과했으며 새 원격 CI는 아직 수행하지 않았다.

## 사용자 결과

완료된 UI는 다음 결과를 보장해야 한다.

- 저장하지 않은 프로필 편집 내용이 어떤 일반 경로에서도 조용히 사라지지 않는다.
- 사용자는 옵션 개수나 UID보다 실제로 적용될 설정값을 먼저 이해할 수 있다.
- 메뉴 막대에서는 가장 중요한 행동인 `적용`이 분명하고, 보조·위험 행동은 의도적으로 분리된다.
- 적용 전 현재 값과 적용 후 값, 위험, 누락, 실행 불가 이유를 친숙한 언어로 확인할 수 있다.
- 준비 상태와 비활성화 이유를 색상이나 마우스 도움말에만 의존하지 않고 이해할 수 있다.
- 영어와 한국어, 키보드, VoiceOver 이름·값, 비색상 상태 단서가 함께 유지된다.
- 프로필 스키마, 어댑터 계약, 적용 순서, 롤백 및 디스플레이 안전 확인 의미는 약화되거나 바뀌지 않는다.

## 디자인 원칙

- macOS 기본 컨트롤, 재질, 간격과 상호작용을 우선하고 별도 디자인 시스템을 만들지 않는다.
- 한 화면에는 하나의 분명한 기본 행동을 둔다.
- 친숙한 이름과 결과를 먼저 보여주고 기술 식별자는 점진적으로 공개한다.
- 위험한 행동은 실행 전에 결과와 복구 방식을 설명한다.
- 빈 상태, 로딩, 비활성, 오류, 부분 성공과 완료 상태를 모두 설계한다.
- 상태는 텍스트와 심볼을 함께 사용하며 색상만으로 표현하지 않는다.
- 단순한 미관보다 편집 내용 보존, 행동 예측 가능성, 오류 회복을 우선한다.

## P0 — 프로필 편집 내용 보호

`ProfilesSettingsView`의 저장본과 편집 초안을 명시적으로 구분하고 다음 상호작용 계약을 구현한다.

- 사용자 편집 필드를 기준으로 clean/dirty 상태를 계산한다.
- 이름, 설명, 아이콘, 활성 상태, 그룹·세부 옵션 포함 여부 및 조건 변경은 dirty 상태가 된다.
- 프로필 선택 변경, 새 프로필, 복제, 삭제, 가져오기 및 현재 설정 갱신처럼 초안을 교체할 수 있는 행동은 dirty 상태를 먼저 해결한다.
- dirty 상태에서 프로필 선택을 바꾸면 `저장`, `변경사항 버리기`, `취소`를 제공한다.
- 취소하면 기존 선택, 초안과 입력 상태가 유지된다.
- 저장은 영속화가 실제로 성공한 뒤에만 clean 상태가 된다.
- 저장 실패 시 초안을 보존하고 사용자가 이해할 수 있는 인라인 오류와 접근성 피드백을 제공한다.
- `model.profiles`가 적용 결과나 저장소 새로 고침으로 변경되어도 dirty 초안을 덮어쓰지 않는다.
- `lastApplication`, 저장 시각처럼 사용자가 직접 편집하지 않는 최신 메타데이터를 오래된 초안으로 덮어쓰지 않는다.
- `현재 설정으로 업데이트`는 계속 읽기 전용이어야 한다. 캡처 결과는 먼저 초안에 반영하고 사용자가 검토·저장하기 전에는 영속 프로필을 교체하지 않는다.
- 저장과 변경 취소 행동은 편집기 스크롤과 무관하게 보이는 영역에 둔다.
- `변경됨`, `저장 중`, `저장됨`, `저장 실패`를 비색상 텍스트와 접근성 값으로 전달한다.
- `⌘S`는 유효한 dirty 초안을 저장하며 저장할 내용이 없을 때는 실행되지 않는다.

## P1 — 적용값 중심의 정보 구조

### 프로필 편집기

- 그룹별 단순 개수 대신 포함된 실제 목표값을 요약한다.
- 가능한 범위에서 다음 값을 표시한다.
  - 디스플레이 이름, 해상도, 새로 고침 빈도, 주 디스플레이와 미러링 상태
  - 오디오 역할, 친숙한 장치명, 볼륨과 음소거
  - Wi-Fi 전원·네트워크, IPv4 방식, DNS와 프록시
  - 포인터 속도, 자연스러운 스크롤, 키 반복과 기능 키 상태
- 제외된 값은 적용될 값처럼 표시하지 않는다.
- 장치 이름을 알 수 없으면 거짓 이름을 만들지 않고 역할 기반 설명을 사용한다.
- UID, 시리얼, 해시형 하드웨어 ID 같은 기술 식별자는 기본 요약에서 숨기고 `기술 정보` disclosure에서만 확인할 수 있게 한다.
- 기존 프로필 모델이 가진 친숙한 메타데이터만 사용하며 이 작업을 위해 프로필 JSON 스키마를 변경하지 않는다.
- 원시 SF Symbol 문자열 입력을 일반 경로에서 제거하고 접근성 이름이 있는 제한된 아이콘 선택기를 제공한다.
- 가져온 프로필의 기존 심볼이 목록에 없어도 값을 임의로 바꾸지 않고 안전한 fallback과 기술 정보를 제공한다.
- 한 개와 여러 개에 맞는 자연스러운 영문·한국어 수량 표현을 사용한다.

### 메뉴 막대 팝오버

- 프로필별 기본 행동은 `적용` 하나로 둔다.
- `적용 가능 여부 확인`과 `사용 가능한 항목만 적용`은 명확한 레이블의 보조 메뉴로 이동한다.
- 강제 적용은 기존 누락 항목 미리보기와 명시적 확인을 그대로 유지한다.
- 적용이 비활성화되면 조건 불충족, 변경 없음, 지원되지 않는 설정, 다른 작업 진행 중 등 이해 가능한 이유를 텍스트와 심볼로 표시한다.
- 프로필 수가 많으면 제한된 높이 안에서 스크롤되며 하단 전역 행동은 계속 접근 가능해야 한다.
- 프로필은 존재하지만 모두 비활성화된 경우 별도 empty state와 Settings 이동 행동을 제공한다.
- 준비 상태 새로 고침 중에는 진행 상태를 표시하고 중복 실행을 방지한다.
- 긴 프로필 이름, 긴 상태 문구와 영어·한국어에서 잘림이나 버튼 겹침이 없어야 한다.

### 적용·가용성 미리보기

- 친숙한 장치명과 행동 설명을 우선하고 원시 식별자는 기본 화면에서 숨긴다.
- `현재`와 `적용 후` 값이 긴 문자열에서도 대응 관계를 잃지 않도록 행 또는 카드 단위로 배치한다.
- 기술 값이 문제 해결에 필요하면 접힌 `기술 정보`에서 선택·복사할 수 있게 한다.
- 내용이 적을 때 과도한 빈 공간이 생기지 않고 내용이 많으면 안전하게 스크롤된다.
- 실행 버튼은 분명한 primary 스타일을 사용하며 취소·닫기와 시각적 계층을 구분한다.
- 읽기 전용 검토에는 실행 버튼을 제공하지 않는다.
- 정상 적용, 강제 적용, 누락, 검증 경고, 고위험 표시와 stale-plan 재확인 흐름을 그대로 유지한다.
- 사용자의 명시적 실행 전에는 어떤 시스템 설정도 변경하지 않는다.

## P2 — 조건, 권한과 신뢰도

### 준비 조건

- 오디오 입출력과 하드웨어 조건은 현재 발견된 친숙한 값의 Picker를 우선 제공한다.
- 저장된 장치가 현재 없으면 `저장된 값 — 현재 연결되지 않음` 상태를 보존한다.
- 기술 식별자 직접 입력이 필요한 호환 경로는 `고급` disclosure 안에 둔다.
- 빈 UID, 빈 SSID, 합성 CIDR 또는 `(0, 0)` 위치를 유효한 기본 조건으로 만들지 않는다.
- 새 조건은 감지된 현재 값으로 채우거나 사용자 입력을 요구하고 저장 전에 인라인 검증한다.
- `조건 집합 반전`과 개별 `아님`은 결과를 예측할 수 있는 자연스러운 영어·한국어 문구로 바꾼다.
- 조건은 자동 적용을 실행하지 않는다는 기존 안내와 의미를 유지한다.
- 위치 권한은 설명 뒤 사용자의 명시적 행동으로만 요청하고 거부는 관련 조건에만 비치명적으로 작동한다.

### 권한

- 로그인 항목의 희망 상태는 `앱 설정`, 실제 결과는 `macOS 등록 상태`로 구분한다.
- 두 값이 일치할 때는 중복 설명을 줄이고, 불일치·승인 필요·오류일 때만 설명 배너와 가능한 재시도 행동을 표시한다.
- 긴 제약 설명은 짧은 기본 설명과 필요 시 추가 정보로 나눠 읽기 부담을 줄인다.
- 권한 거부나 로그인 항목 등록 실패가 관련 없는 프로필 편집·검토 기능을 막지 않는다.

### 앱 정보와 진단

- About에 저장소, 문제 신고, MIT 라이선스와 개인정보 원칙으로 이동하는 명확한 링크를 제공한다.
- 외부 네트워크 접근은 사용자가 링크를 클릭했을 때만 시스템 브라우저에서 발생한다.
- 진단 심각도는 심볼뿐 아니라 `정보`, `경고`, `오류` 같은 텍스트와 접근성 레이블을 함께 제공한다.
- 진단 화면의 전면 개편은 이번 목표에 포함하지 않는다.

## 접근성 계약

- 모든 아이콘 전용 버튼은 영문·한국어 접근성 이름과 도움말을 갖는다.
- 키보드만으로 메뉴, 프로필 선택, 편집, 저장·취소, 보조 메뉴와 대화상자를 사용할 수 있어야 한다.
- 디스플레이 안전 확인의 `지금 되돌리기`는 Escape/cancel 동작이고 `변경사항 유지`는 default 동작이다.
- 안전 확인 진행 막대는 남은 초를 최신 accessibility value로 제공한다.
- 중요한 저장·적용 실패와 상태 전환은 작은 보조 문구만 갱신하지 않고 포커스를 뺏지 않는 접근성 알림 또는 의미 있는 상태 영역으로 전달한다.
- Ready, Partial, Unavailable, Applying과 오류 상태는 텍스트와 심볼을 함께 사용한다.
- 영어·한국어, 긴 값, 최소 창 크기와 macOS 접근성 표시 설정에서 정보가 잘리거나 행동이 사라지지 않아야 한다.
- 실제 VoiceOver 발화 품질과 포커스 순서는 가능한 범위에서 수동 확인하고, 수행하지 않은 항목은 검증했다고 주장하지 않는다.

## 현지화

- 모든 새 사용자 문구, 접근성 이름·힌트, 오류와 상태 문구에 영어와 한국어 값을 제공한다.
- 두 언어의 키 집합, 중복 키와 format placeholder 호환성을 자동 검사한다.
- `Availability Review`, `Force Apply`, `Capture Current Settings`, 로그인 상태와 조건 반전 문구를 사용자의 결과가 드러나는 자연스러운 표현으로 개선한다.
- 단수·복수와 한국어 수량 표현을 테스트한다.
- 원시 Core 오류가 한국어 UI에 그대로 노출되지 않도록 typed 오류를 사용자 문구로 매핑한다.

## 아키텍처와 안전 경계

- SwiftUI/AppKit은 앱 레이어에 둔다.
- 편집 초안, 표시 요약과 행동 상태처럼 결정론적으로 검증해야 하는 로직은 시스템 프레임워크와 분리된 순수 타입으로 만든다.
- Core의 프로필·조건·트랜잭션 의미와 어댑터 프로토콜을 UI 편의를 위해 약화하지 않는다.
- UI는 `ApplicationModel`과 Core 프로토콜을 통해 동작하고 시스템 mutation을 직접 호출하지 않는다.
- `snapshot`, `validate`, `plan`, `apply`, `rollback`, `capability`, `diagnostics` 의미를 보존한다.
- 가져온 JSON은 계속 신뢰하지 않는 입력으로 검사한다.
- 비밀번호나 비밀정보를 프로필, fixture, 로그 또는 화면 증거에 저장하지 않는다.
- 실제 SSID, 정확한 위치, IP, 장치 UID, 시리얼과 개인 식별자를 fixture, 커밋 또는 스크린샷 증거에 넣지 않는다.
- 공개 API, 최소 권한, 로컬 우선과 비치명적 권한 거부 원칙을 유지한다.
- telemetry, analytics, cloud, 임의 shell 실행, UI 자동화, private API 또는 서드파티 설정 파일 변경을 추가하지 않는다.

## 테스트와 검증

시스템 설정을 변경하지 않는 결정론적 테스트를 추가한다.

### 편집 초안

- 최초 선택은 clean이다.
- 이름, 설명, 아이콘, 토글, 옵션과 조건 변경은 dirty다.
- 저장 성공 후 clean, 변경 버리기 후 원본 복구, 취소 후 선택·초안 유지가 성립한다.
- dirty 상태의 선택 변경에서 저장·버리기·취소가 각각 올바르게 동작한다.
- 저장 검증 또는 영속화 실패가 선택을 이동하거나 초안을 잃게 하지 않는다.
- 외부 프로필 갱신이 dirty 초안을 덮어쓰지 않는다.
- 삭제, 복제, 가져오기와 현재 설정 갱신도 같은 보호 규칙을 거친다.
- 현재 설정 캡처 결과는 시스템을 변경하지 않고 초안만 dirty로 만든다.

### 표시 모델

- 디스플레이, 오디오, 네트워크와 입력 그룹 요약이 포함 여부와 안정적인 순서를 지킨다.
- nil, unsupported, 긴 이름, 여러 장치와 친숙한 이름 부재를 처리한다.
- 기술 식별자는 기본 요약의 주 정보가 되지 않는다.
- Ready, Partial, Unavailable, Applying, zero-operation, disabled profile과 transaction lock별 행동과 비활성 이유를 생성한다.
- normal, force, review-only, zero-operation과 high-risk 미리보기를 구분한다.
- 실제 개인정보 대신 합성·비식별 fixture만 사용한다.

### 현지화와 회귀

- 영문·한국어 키 집합, 중복 키와 placeholder 호환성 검사를 자동화한다.
- 새 앱 레이어 순수 로직은 전용 테스트 타깃 또는 동등한 테스트 가능한 구조에서 검증한다.
- 기존 적용 계획, recapture, rollback과 어댑터 테스트를 모두 유지한다.
- 구현 마일스톤마다 최소 `make test`, `make lint`, `git diff --check`를 실행한다.
- 최종 통합 트리에서 `make verify`와 `git diff --check`를 통과한다.

## 읽기 전용 UI 증거

가능하면 합성·비식별 프로필이 있는 격리 환경에서 다음을 영어와 한국어로 확인한다.

- 프로필 0개, 1개, 여러 개와 모두 비활성화 상태
- Ready, Partial, Unavailable, Applying과 변경 없음 상태
- 긴 프로필 이름, 긴 설명과 긴 장치 표시명
- 메뉴 팝오버, 프로필 편집, 가용성 검토, 적용 미리보기, 권한과 About
- 최소·기본·넓은 Settings 크기의 스크롤, 잘림과 버튼 겹침
- Tab/Shift-Tab, Space/Return, `⌘,`, `⌘S`와 Escape
- dirty 초안의 저장·버리기·취소
- Increase Contrast, Reduce Transparency와 가능한 텍스트 크기 설정

읽기 전용 검토에서도 `적용`, `강제 적용`, 현재 설정 mutation 또는 실제 권한 요청을 실행하지 않는다. 화면 증거에는 합성 데이터만 사용하고 임시 캡처를 기본으로 한다. 저장소에 자산을 추가해야 한다면 출처와 라이선스를 문서화한다.

다음 항목은 자동 테스트만으로 증명할 수 없으며 수행 여부를 별도로 기록한다.

- 실제 VoiceOver 발화 품질과 전체 포커스 순서
- 실제 디스플레이 변경 뒤 안전 카운트다운 발화와 Escape 롤백
- 실제 TCC 승인·거부, 로그인 항목 승인·재시도·재부팅
- 물리 Intel, 외장 디스플레이와 다양한 실제 장치명
- display, audio, network, mouse, keyboard mutation과 rollback
- 실제 Keychain write

## 문서와 증거

동작 또는 검증 상태가 바뀌면 다음을 실제 증거에 맞게 갱신한다.

- `README.md`: 새 메뉴·편집 흐름, 단축키와 최종 검증 수치
- `docs/ROADMAP.md`: M4 UI hardening 상태와 다음 미검증 작업
- `docs/SUPPORT-MATRIX.md`: UI가 표현하는 capability와 실제 검증 경계
- `docs/COMPLETION-CRITERIA.md`: 자동, 수동과 하드웨어 증거 분리
- `CHANGELOG.md`: 사용자에게 보이는 동작 변경
- 필요 시 `docs/PRODUCT.md`, `docs/ARCHITECTURE.md`, `docs/TECHNICAL-SPEC.md`

완료하지 않은 VoiceOver, 권한 또는 하드웨어 항목을 체크하지 않는다.

## 작업 순서와 커밋

1. 이 목표와 현재 UI 기준선을 저장소에 기록한다.
2. 초안 상태와 표시 모델을 순수 타입으로 설계하고 결정론적 테스트를 먼저 추가한다.
3. 초안 손실 방지와 고정 저장·취소 흐름을 구현하고 검증한다.
4. 메뉴 행동 계층, 프로필 요약과 적용 미리보기를 구현하고 검증한다.
5. 조건, 권한, About, 접근성과 현지화를 구현하고 검증한다.
6. 읽기 전용 화면 검토와 전체 `make verify`를 수행한다.
7. README, roadmap, support matrix, completion ledger와 changelog에 실제 증거를 기록한다.
8. 의미 있는 검증 단위로 커밋하고 `origin`에 push한다.
9. 동일한 최종 SHA의 GitHub Actions와 unsigned artifact upload가 성공할 때까지 확인한다.

권장 구현 커밋은 다음과 같다. 실제 변경 경계에 따라 합리적으로 조정할 수 있다.

- `feat: protect unsaved profile edits`
- `feat: clarify profile and apply workflows`
- `docs: record UI refinement evidence`
