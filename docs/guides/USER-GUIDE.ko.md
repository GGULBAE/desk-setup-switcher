# Desk Setup Switcher 사용자 가이드

[English](USER-GUIDE.md) · [가이드 인덱스](README.md)

Desk Setup Switcher는 선택한 디스플레이·오디오·네트워크 설정을 로컬 프로필로 저장합니다. 어떤 프로필도 자동으로 적용하지 않습니다. 기본 흐름은 **Capture → Edit → Review & Apply**입니다.

## 설치 전 확인

아직 지원되는 공개 다운로드가 없습니다. 저장소와 CI에서 만든 DMG는 ad-hoc 서명된 개발 검증 자료이며, 서명·공증된 일반 사용자용 릴리스가 아닙니다. 일반 사용자는 Gatekeeper를 우회해 설치하지 마세요.

`v0.1.0`이 승인되어 공개된 뒤에는 프로젝트의 [GitHub Releases 페이지](https://github.com/GGULBAE/desk-setup-switcher/releases)가 유일한 공식 다운로드 경로가 됩니다. 초기 public beta 예정 환경은 다음과 같습니다.

- Apple Silicon
- exact candidate가 Sonoma 수명주기 검증을 통과한 뒤의 macOS 14 Sonoma deployment target
- 공개된 GitHub Release에 첨부된 서명·공증 DMG

DMG에 Intel slice가 들어 있더라도 Intel 지원을 의미하지 않습니다. 실제 Intel Mac 설치·실행 검증은 아직 통과하지 않았습니다. App Store 릴리스, 앱 내 업데이트, 지원되는 Homebrew 설치도 아직 없습니다.

2026-07-20 현재 소스에서 opt-in 읽기 전용 테스트의 Display, Audio, Network, Input, ConditionContext, ApplyLivePreparation 그룹/기본 경로가 Apple Silicon/macOS 26.5.2에서 통과했습니다. 다만 이 테스트는 해당 호스트의 ColorSync 프로필·입력 볼륨·서비스 IPv4 개별 필드 존재/읽기를 항목별로 확인하지 않았습니다. 따라서 그 개별 실기 읽기 주장과 모든 Display, Audio, Network 적용·rollback 경로는 mock 검증 전용으로 남으며, 실제 설정 변경은 검증되지 않았습니다. 기능에 의존하기 전에 [지원표](../SUPPORT-MATRIX.md)를 확인하세요. 이 가이드는 하드웨어 인증을 의미하지 않습니다.

키보드 동작, 접근성 이름·값, 비색상 상태 단서는 유지합니다. 포괄적인 보조 기술 인증은 초기 베타 범위에 포함하지 않습니다.

## 정식 공개 후 설치

1. 공개된 GitHub Release의 노트가 unsigned candidate, 중단 또는 문제 있는 릴리스가 아닌 승인된 public beta라고 명시하는지 확인합니다. public beta에는 GitHub의 **Pre-release** 배지가 표시될 수 있습니다.
2. 같은 Release에서 DMG와 공개된 checksum을 내려받고 SHA-256이 릴리스 기록과 일치하는지 확인합니다.
3. DMG를 열고 **Desk Setup Switcher**를 **응용 프로그램**으로 드래그한 뒤 DMG를 추출합니다.
4. **응용 프로그램**에서 앱을 엽니다. 인터넷에서 받은 앱에 대한 일반적인 최초 실행 확인은 정상입니다. 개발자를 확인할 수 없거나, 앱을 검사할 수 없거나, 파일이 손상됐다는 메시지가 나오면 중단하세요. 공식 릴리스에 **확인 없이 열기(Open Anyway)**를 사용하지 말고 다운로드와 checksum을 확인한 뒤 신고하세요.
5. 메뉴 막대에서 Desk Setup Switcher 아이콘을 찾습니다. 메뉴 막대 전용 앱이므로 실행할 때 Dock 아이콘이나 메인 창이 보이지 않는 것이 정상입니다.

**로그인 시 실행은 기본으로 꺼져 있습니다.** 필요할 때만 **설정 → 시스템 → 로그인**에서 켜세요. macOS 승인이 필요할 수 있으므로 앱은 사용자가 요청한 값과 macOS 등록 상태를 따로 보여줍니다.

## 1분 사용 흐름

1. **Capture:** 메뉴 막대 항목을 열고 **현재 설정 캡처**를 선택합니다. Capture는 Mac의 현재 값을 읽어 검토할 새 프로필을 만들며 설정을 적용하지 않습니다.
2. **Edit:** 프로필의 **프로필 수정**을 선택하고 이름을 지정한 뒤 **디스플레이**, **오디오**, **네트워크**에서 필요한 설정만 골라 저장합니다.
3. **Review & Apply:** **검토…** 또는 **사용 가능한 항목 검토…**를 선택합니다. 변경과 제외 항목을 모두 확인하세요. **프로필 적용** 또는 **사용 가능한 설정 적용**을 명시적으로 누르기 전에는 어떤 설정도 바뀌지 않습니다.

타이머, 조건 또는 백그라운드 규칙으로 프로필을 자동 적용하는 기능은 없습니다.

## 1. Capture

메뉴 막대 항목을 열고 **현재 설정 캡처**를 선택합니다. 프로필이 없는 최초 상태에서는 본문의 유일한 주 동작이며, 프로필이 있으면 헤더에 표시됩니다.

Capture는 형식 호환성을 위해 읽을 수 있는 snapshot 값을 로컬 프로필에 기록합니다. 현재 앱이 capture, 검증, 계획, 적용, 확인, rollback 계약을 끝까지 수행할 수 있는 값만 편집기에서 실행 가능한 항목으로 표시합니다. 지원되지 않거나 읽을 수 없는 값은 실행 가능해지지 않으며, dormant 또는 snapshot 전용 값은 포함이 꺼진 채 저장되어 내보낸 JSON에 남을 수 있습니다.

macOS에서는 현재 Wi-Fi 네트워크 이름을 확인할 때 위치 접근이 필요할 수 있습니다. 앱은 권한을 요청하기 전에 이유를 설명합니다. 다음 중 하나를 선택할 수 있습니다.

- **위치 접근 허용 후 캡처**
- **Wi-Fi 없이 캡처**
- macOS 시스템 설정을 연 뒤 앱으로 돌아와 다시 Capture

위치 접근을 거부해도 디스플레이, 오디오, 유선 네트워크 등 관련 없는 값은 계속 사용할 수 있습니다. 이 권한은 macOS가 현재 Wi-Fi 이름을 제공할 수 있게 하는 데만 사용되며, 앱은 좌표를 요청하거나 위치 데이터를 업로드하지 않습니다.

## 2. Edit

프로필의 **프로필 수정**을 선택하면 **설정 → 프로필**이 열립니다.

- 알아보기 쉬운 프로필 이름과 아이콘을 지정합니다.
- 각 설정의 **프로필 적용에 포함**을 사용해 프로필 포함 여부를 정합니다. **포함됨**과 **포함되지 않음**은 색상뿐 아니라 텍스트와 기호로 표시됩니다.
- 실제로 바꾸려는 값만 설정합니다. 현재 편집기는 Mac이 제공하는 capability에 따라 지원되는 디스플레이 모드·주 디스플레이·ColorSync, 오디오 기본 장치와 조절 가능한 볼륨, 정확한 Ethernet 또는 Wi-Fi 서비스의 DHCP/수동 IPv4를 보여줍니다.
- 프로필을 저장합니다. 유효한 변경 초안은 `⌘S`로 저장할 수 있습니다.

저장된 대상 장치가 사라진 경우 가능한 범위에서 경고와 함께 계속 표시됩니다. 장치를 다시 연결하거나 해당 행의 **프로필 적용에 포함**을 끈 뒤 저장하세요. 선택지가 보이지 않는다고 지원된다고 추정하면 안 됩니다. 앱은 현재 snapshot에서 capture/apply/verify/rollback 계약을 끝까지 수행할 수 없는 새 선택지를 숨깁니다.

저장하지 않은 변경이 있을 때 다른 프로필 선택, 가져오기, 적용 또는 종료를 시도하면 저장·버리기·취소 중 하나를 묻습니다. 저장 실패 시 초안은 유지됩니다. Capture는 별도 프로필을 만들며 열려 있는 초안을 몰래 덮어쓰지 않습니다.

## 3. Review & Apply

각 프로필에는 상태에 맞는 검토 동작 하나가 있습니다.

- **검토…**: 포함된 적용 가능 설정을 모두 준비할 수 있습니다.
- **사용 가능한 항목 검토…**: 실행 가능한 항목은 있지만 일부 항목이 제외·차단·미지원·사용 불가 상태입니다.

검토 화면은 읽기 전용입니다. 계속하기 전에 대상 값, 작업 목록, 제외 항목과 기술 상세를 확인하세요.

- **프로필 적용**은 완전한 계획을 시작합니다.
- **사용 가능한 설정 적용**은 나열된 실행 가능 항목만 적용합니다. 모든 제외가 의도한 것일 때만 사용하세요.
- 검토 화면을 연 뒤 Mac 상태가 바뀌면 앱은 아무것도 적용하지 않고 갱신된 검토를 다시 보여줍니다. 새 계획을 다시 확인하세요.
- 이미 현재 상태와 같은 프로필은 실행할 변경이 없어 적용할 수 없습니다.

실행 후에는 항목별 결과를 확인하세요. 각 항목은 성공, 실패, 건너뜀, 미지원, rollback 완료, rollback 실패 또는 검증 안 됨으로 표시될 수 있습니다. 성공 표시는 read-back까지 통과해야 하며, 다시 읽을 수 없거나 변경이 여전히 필요해 보이면 **검증 안 됨**입니다.

### 보호 변경과 rollback

위험도가 높은 디스플레이와 네트워크 변경은 15초 안전 창이 열려 있는 동안 임시 상태로 유지됩니다.

- Mac을 정상적으로 사용할 수 있고 결과가 검토 내용과 같을 때만 **변경 사항 유지**를 선택하세요.
- 디스플레이, 네트워크 또는 결과가 예상과 다르면 **지금 되돌리기**를 선택하세요.
- 안전 창을 닫거나 앱을 종료하거나 시간이 끝나면 이전 구성을 복원하도록 요청합니다.

치명적인 실패가 발생하면 트랜잭션 엔진이 rollback을 지원하는 완료 항목을 역순으로 되돌리려 시도합니다. rollback은 capability마다 다르고 rollback 자체도 실패할 수 있습니다. 항목별 결과와 현재 macOS 설정을 직접 확인해야 합니다. 보호 변경 또는 rollback이 안전하지 않게 동작하면 가능한 경우 macOS 시스템 설정에서 원래 값을 복구하고 현재 [SECURITY.md](../../SECURITY.md) 안내를 따르세요. 비공개 취약점 신고 기능은 아직 비활성 상태이므로 첫 연락이나 공개 이슈에 민감한 상세를 넣지 말고 비공개 채널을 요청하세요.

## 권한과 개인정보

Desk Setup Switcher는 로컬 전용입니다. 계정, 클라우드 동기화, 앱 소유 서버, 텔레메트리, 분석, 광고 또는 자동 프로필 전환이 없습니다.

| 접근 | 표시될 수 있는 이유 | 거부할 경우 |
| --- | --- | --- |
| 위치 | Capture 중 현재 Wi-Fi 이름을 확인하기 위해 macOS가 요구할 수 있음 | Wi-Fi 없이 Capture 가능, 관련 없는 기능은 계속 동작 |
| 보호된 네트워크 변경에 대한 macOS 승인 | 포함된 서비스별 IPv4 변경에 승인된 SystemConfiguration 접근이 필요할 수 있음 | 취소하거나 거부하면 해당 항목은 실패/미적용 |
| 마이크 | 사용하지 않음. 오디오 입력 장치를 선택해도 녹음하지 않음 | 마이크 권한이 필요하지 않아야 함 |

프로필과 진단은 `~/Library/Application Support/Desk Setup Switcher/` 아래에 남습니다. 가져오기는 사용자가 선택한 파일을 읽고, 내보내기는 사용자가 선택한 위치에 씁니다. 앱은 Wi-Fi 비밀번호를 프로필이나 로그에 저장하지 않으며, 저장된 네트워크 자격 증명은 macOS가 관리합니다.

전체 로컬 데이터 경계는 [개인정보 보호 정책](../PRIVACY.md)을 확인하세요.

## 가져오기와 내보내기

**설정 → 프로필**을 열고 **추가 프로필 작업** 메뉴를 엽니다.

### 내보내기

1. 내보낼 변경 초안을 먼저 저장합니다.
2. **저장된 프로필 내보내기…**를 선택합니다.
3. 새 로컬 `.json` 위치를 선택합니다.

내보내기는 저장된 모든 프로필만 포함하며 현재 저장하지 않은 초안은 포함하지 않습니다. 기존 목적지 파일을 덮어쓰지 않습니다. JSON에는 비밀번호가 없지만 프로필·장치 이름, SSID, 네트워크 범위, 안정 식별용 기기 정보, 정확한 legacy 위치 조건이 포함될 수 있습니다. Dormant 또는 snapshot 전용 값은 포함이 꺼진 채 남을 수 있습니다. 공유하기 전에 파일을 직접 검토하고 가리세요.

### 가져오기

1. **가져오기…**를 선택하고 로컬 `.json` 프로필 문서 하나를 고릅니다.
2. 검증 결과와 교체될 프로필 수를 확인합니다.
3. 로컬 프로필 **전체**를 교체하려는 경우에만 확인합니다. 앱은 마지막 정상 backup을 유지합니다.

가져오기는 병합이 아니라 전체 교체입니다. 앱은 유효하지 않거나, 너무 크거나, 안전하지 않거나, 실행 앱보다 새 schema이거나, ID가 중복된 문서를 저장 전에 거부합니다. 오래된 schema는 migration될 수 있지만 가져온 legacy 조건은 dormant 상태로 남아 수동 흐름을 시작하거나 막지 않습니다. JSON 형식은 앱 간 교환과 복구용이며 안정적인 자동화 API가 아닙니다. [호환성과 버전 정책](../COMPATIBILITY.md)을 참고하세요.

## 진단과 지원

Capture, 준비 상태, 저장소 또는 Apply 문제가 생기면 다음을 수행합니다.

1. **설정 → 시스템 → 고급 진단 열기…**를 선택합니다.
2. 최근 민감 정보 제거 이벤트를 보려면 **새로 고침**을 선택합니다. **준비 상태 새로 고침**은 현재 사실을 읽기 전용으로 다시 확인합니다.
3. 마지막 결과, 마지막 snapshot, 준비 상태 사실과 항목별 상태를 확인합니다.
4. **이벤트 지우기…**를 선택하면 앱이 순환 저장한 진단 이벤트 파일만 삭제됩니다.

진단은 로컬에만 있으며 자동 업로드되지 않습니다. 진단 내보내기는 구현되어 있지 않습니다. 저장 이벤트는 민감 정보 제거 과정을 거치지만 진단 화면과 프로필 내보내기에는 현재 식별자나 사용자가 선택한 값이 보일 수 있습니다. 도움을 요청하기 전에 비밀번호, 일련번호, 실제 SSID, 정확한 위치, IP 호스트 부분, 기기 UID, 홈 경로, 검토하지 않은 스크린샷을 제거하세요.

일반 지원과 버그 신고 경로는 [SUPPORT.md](../../SUPPORT.md)를 이용하세요. 취약점, 안전하지 않은 변경, 개인정보 유출, 노출된 비밀 또는 rollback 실패는 현재 [SECURITY.md](../../SECURITY.md) 안내를 따르세요. 비공개 신고 기능은 아직 비활성 상태이므로 상세를 공개하지 말고 비공개 채널을 먼저 요청하세요.

## 문제 해결

| 증상 | 해결 방법 |
| --- | --- |
| 앱을 열었지만 창이 보이지 않음 | 메뉴 막대 아이콘을 찾으세요. 메뉴 막대 전용 앱이므로 정상 동작입니다. |
| Wi-Fi 이름이 캡처되지 않음 | 설명을 확인한 뒤 위치 접근을 허용하거나 **Wi-Fi 없이 캡처**를 명시적으로 선택하세요. 시스템 설정에서 권한을 바꾼 뒤 앱으로 돌아와 다시 Capture하세요. |
| 프로필이 일부 사용 가능 또는 사용 불가임 | **수정**에서 경고 행을 확인하고 필요한 장치·서비스를 다시 연결하거나 사용 불가 설정을 제외하세요. 모든 제외가 의도한 경우에만 **사용 가능한 항목 검토…**를 사용하세요. |
| 검토 화면에 작업이 없음 | 프로필이 현재 읽을 수 있는 상태와 이미 같거나 포함된 설정을 안전하게 적용할 수 없습니다. 실행할 작업이 없습니다. |
| 적용 대신 검토 화면이 갱신됨 | 첫 검토 후 프로필, capability, 현재 값 또는 rollback 근거가 바뀌었습니다. 안전을 위한 중단이므로 새 계획을 확인하세요. |
| 프로필 저장소 오류가 표시됨 | 로드 실패에는 **다시 불러오기**, 일반 작업 실패 뒤에는 **오류 닫기**를 사용하세요. 앱 실행 중 관리 파일을 직접 수정하지 마세요. 필요하면 검토한 내보내기·가져오기로 복구하세요. |
| 보호 변경 후 Mac을 사용할 수 없음 | 15초가 끝나기 전에 **지금 되돌리기**를 선택하세요. 이후 macOS 시스템 설정에서 원래 상태와 항목별 결과를 확인하세요. |
| 공식 DMG가 Gatekeeper를 통과하지 못함 | 중단하고 Gatekeeper를 끄거나 **확인 없이 열기**를 사용하지 마세요. Release URL과 checksum을 확인한 뒤 민감 정보 없이 정확한 메시지를 신고하세요. |
| 로그인 시 실행을 켰지만 등록되지 않음 | **설정 → 시스템**에서 요청 값과 **macOS 등록**을 비교하고, 필요하면 macOS 로그인 항목에서 승인한 뒤 **상태 새로 고침** 또는 **등록 다시 시도**를 선택하세요. |

## 업데이트

앱은 네트워크에서 업데이트를 확인하지 않습니다. 공개 릴리스가 생긴 뒤에는 공식 GitHub Releases 페이지에서 새 버전을 직접 받으세요. 기존 앱을 ad-hoc CI artifact로 교체하지 마세요. 릴리스 노트가 profile migration 또는 복구 위험을 알리면 먼저 저장된 프로필을 내보내세요.

## 앱 제거와 로컬 데이터 삭제

앱을 제거해도 프로필과 진단은 자동 삭제되지 않습니다.

1. 켜져 있다면 **설정 → 시스템 → 로그인할 때 책상 설정 전환기 실행**을 끄고 **macOS 등록 상태**도 비활성 상태인지 확인합니다.
2. 메뉴 막대 앱에서 **책상 설정 전환기 종료**를 선택합니다.
3. **응용 프로그램**의 **Desk Setup Switcher.app**을 휴지통으로 옮깁니다.
4. 프로필, backup/quarantine 파일과 진단까지 삭제하려면 Finder의 **이동 → 폴더로 이동…**에서 `~/Library/Application Support/Desk Setup Switcher/`를 열어 삭제합니다. 보관할 프로필은 먼저 내보내세요.
5. 환경설정까지 완전히 초기화하려면 앱이 종료된 상태에서 `~/Library/Preferences/dev.ggulae.desk-setup-switcher.plist`도 삭제합니다.
6. 위치 접근을 허용했다면 선택적으로 **시스템 설정 → 개인정보 보호 및 보안 → 위치 서비스**에서 Desk Setup Switcher를 끕니다.

다른 위치에 내보낸 프로필 JSON은 앱이 추적하거나 삭제하지 않으므로 원하면 별도로 삭제하세요. 필요한 내보내기 파일이 없는지 확인한 뒤 휴지통을 비우세요. 현재 앱은 Wi-Fi 비밀번호를 요청하거나 저장하지 않으므로 앱 파일을 삭제해도 macOS가 관리하는 네트워크 자격 증명은 삭제되지 않습니다.
