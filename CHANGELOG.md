### Changed

- Apply Preview now uses one review-to-decision scroll flow. A localized text-and-shield Beta status says Apply and rollback are not hardware-verified and asks the user to check System Settings afterward; decision actions follow the review content, and Apply is no longer a default Return-key action.
- Legacy-workflow containment now owns a closed set of fixed GitHub operations and hashes response bytes while they stream, then binds each capture to a caller-held HMAC attestation and the apply request to the reviewed plan digest. Repository policy/library code is evaluated only from no-follow descriptor reads that match the recorded Git blobs; the secret bundle crosses an anonymous stdin pipe; `gh` uses the exact root-owned empty `/private/var/empty` configuration directory and a minimal telemetry/update/prompt-disabled child environment; and cancellation from the first disable byte through final receipt remains exit 75. Same-inode truncation, post-validation code replacement, runtime/config injection, forged attestation, plan-authorization, and signal regressions are covered under the documented trusted-local-toolchain threat boundary.
- Legacy-workflow containment pagination now works with GitHub CLI 2.95 without combining incompatible `--slurp` and `--jq` flags. Paginated tag, Release, and workflow-run projections are emitted as minimal JSON lines and strictly canonicalized by the policy; malformed, duplicate-key, inconsistent-total, truncated, and multi-page cases are covered without retaining unprojected response fields. The helper also canonicalizes a trailing-slash `TMPDIR` before its absolute-path checks.
- Holding and published site states now use distinct English/Korean hero, support, and private-reporting copy; the published render no longer retains candidate/planned-gate language. Production metadata includes a validated canonical URL and `og:url`, the Korean home-link accessibility name is localized, and the immutable curated Release body is self-contained: it rejects relative/branch document links and permits only the tag-pinned distribution procedure plus exact support/advisory action routes.
- The v3 remote-release contract now requires both the macOS app and public-site/release-asset CI checks in the final `master` ruleset, collects both check runs and both workflow jobs from one successful check suite, and rejects missing, failed, cross-bound, or different-suite evidence across predecessor-pre-tag, final-pre-tag, and pre-publication phases. The final publication helper applies the same exact two-job requirement to its approval commit. Legacy v1/v2 fixture contracts remain regression coverage.
- Managed profile replacement no longer re-resolves the parent path for production mutations. Post-commit parent/leaf identity checks fail closed and restore the pre-commit leaf when that restoration can be proven safe. Darwin provides no public inode-conditional rename or unlink, so a noncooperative process under the same UID can still race the final adjacent metadata-check and mutation syscalls. An abrupt stop between existing-leaf swap and old-leaf unlink can also leave the former private `0600` file under its randomized staging name. These residual kernel-CAS, crash-residue, and best-effort directory-durability boundaries are documented rather than presented as absolute guarantees.
- Profile export now writes and synchronizes a short randomized owner-private `0600` sibling staging file before an exclusive `RENAME_EXCL` commit. Existing entries win without cooperative replacement, and the user-selected parent stays descriptor-anchored. Export accepts current-user-owned parents without shared POSIX writes, sticky current-user-owned parents, and sticky root-owned compatibility parents such as macOS `/tmp`; it rejects other owners, non-sticky shared-write parents, and every extended-ACL ALLOW entry while permitting deny-only ACLs. App-managed directories/files and new staging files have extended ACLs removed and absence-verified before private mode enforcement. Deterministic ACL, destination-race, special-entry, parent/staging replacement, maximum-leaf-length, and pre/post-commit regressions verify complete-byte publication and identity-guarded cleanup under the documented same-UID race boundary.
- Release signing import now rejects an ephemeral Keychain unless it contains exactly one valid codesigning identity in total and that identity exactly matches the reviewed Developer ID Application label; synthetic extra-valid, missing-match, malformed-inventory, and cleanup regressions cover the fail-closed boundary.
- Every ordinary failure after `PATCH_ATTEMPT_BEGIN` now emits `INCIDENT_ONLY_FAILURE`, while pre-PATCH failures, pre-existing public state, forbidden reruns, and successful publication retain distinct classifications.
- The opt-in display live-read regression now counts optional ColorSync snapshot items separately from active display topology, with deterministic coverage for a display that contributes one portable ColorSync item.
- The release-evidence lifecycle now has one unambiguous linear predecessor/final history: protected predecessor commit P, add-only predecessor evidence P1, protected final commit F, and add-only final evidence F1, with direct annotated tags binding the two evidence digests. P1 and F1 must each occur exactly once on the required first-parent path; merge-only or second-parent integration is rejected. Separate tag-push approvals precede the first push of either recorded tag object, and only the final tag can create a draft. Canonical UTC evidence must satisfy the phase-specific manual/evidence/approval chronology. Publication-side `HUP`, `INT`, `QUIT`, `TERM`, cancellation, runner/host loss, and unavailable/incomplete logs are incident-only rather than retry evidence.
- Publication credential documentation now distinguishes the signing/notarization secrets' single direct consumers from the admin-read token's exactly six bounded workflow secret references and five tracked publication-helper read/download call sites. The ordinary write token remains confined to the single exact-ID PATCH.
- Profile decoding now fails closed when `schemaVersion` is missing, Boolean, fractional, or floating-point JSON while preserving the explicit schema 0→1 migration. Included Display ColorSync settings with no value are rejected instead of reaching planning.
- Snapshot coordination now rejects capability-group mismatches without ingesting or exposing the mismatched payload, while continuing unrelated groups. New adapter-contract guidance records this fail-closed boundary.
- The unused coordinate reader/cache and location-refresh Settings surface are removed. Location authorization is requested only at explicit Capture for the current Wi-Fi SSID; no coordinates are requested or stored, and imported legacy location conditions remain dormant compatibility data.
- Protected-change copy now describes rollback as a best-effort restore attempt rather than a guarantee, matching the adapter and transaction contract.
- The site production-origin gate now rejects loopback, IP-literal, trailing-dot, single-label, reserved, credential-bearing, and path/query/fragment origins; only explicitly flagged HTTP loopback builds are accepted for local verification. Built-client storage and generated Worker bindings are deny-by-default regression gates.
