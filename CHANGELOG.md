### Changed

- The final-pre-tag evidence lifecycle now has one unambiguous linear history: clean A and exact-A CI, external mode-0600 evidence E, a local unpushed annotated tag object targeting A and binding E, direct single-parent add-only B, exact-B CI plus A→B semantic recheck, separate tag-push approval, and only then the first push of the recorded tag object and draft build. Merge commits are forbidden for A→B. Canonical UTC evidence must satisfy `final manuals observedAt ≤ final E collectedAt ≤ pre manuals observedAt ≤ pre manifest collectedAt ≤ approval approvedAt`. Publication-side `HUP`, `INT`, `QUIT`, `TERM`, cancellation, runner/host loss, and unavailable/incomplete logs are incident-only rather than retry evidence.
- Publication credential documentation now distinguishes the signing/notarization secrets' single direct consumers from the admin-read token's exactly five bounded workflow uses and five tracked publication-helper read/download call sites. The ordinary write token remains confined to the single exact-ID PATCH.
- Profile decoding now fails closed when `schemaVersion` is missing, Boolean, fractional, or floating-point JSON while preserving the explicit schema 0→1 migration. Included Display ColorSync settings with no value are rejected instead of reaching planning.
- Snapshot coordination now rejects capability-group mismatches without ingesting or exposing the mismatched payload, while continuing unrelated groups. New adapter-contract guidance records this fail-closed boundary.
- The unused coordinate reader/cache and location-refresh Settings surface are removed. Location authorization is requested only at explicit Capture for the current Wi-Fi SSID; no coordinates are requested or stored, and imported legacy location conditions remain dormant compatibility data.
- Protected-change copy now describes rollback as a best-effort restore attempt rather than a guarantee, matching the adapter and transaction contract.
- The site production-origin gate now rejects loopback, IP-literal, trailing-dot, single-label, reserved, credential-bearing, and path/query/fragment origins; only explicitly flagged HTTP loopback builds are accepted for local verification. Built-client storage and generated Worker bindings are deny-by-default regression gates.
- Launch at login is now off until the user explicitly opts in. Existing pre-release state that could have been created by the former automatic registration path is reset and any stale registration is removed; a consent-version marker preserves subsequent explicit on/off choices.
- Profile load, save, recovery, migration, quarantine, and import now fail closed on unsafe paths or changing file identities while keeping unrelated permission denial nonfatal and user-facing storage errors sanitized.
- The local proposed signed-candidate workflow is now a manual exact-tag/commit dispatch scoped to the unconfigured `release-candidate` environment. It is designed to prepare a Developer ID/notarized/stapled draft prerelease, SBOM/attestations, draft redownload verification, and a protected workflow artifact without a public publish command, but it is unpushed and unproven with credentials. The effective remote workflow remains an unsafe historical unsigned-publication path, so no release tag may be pushed until that path is removed and protected remote controls are proven.
- Release-tag protection is split so the exact release operator can bypass `v*` creation only; a separate no-bypass rule blocks update and deletion. Creation authority no longer implies authority to move or delete an existing release tag.
- Release credential handling now rejects self-hosted runners, scopes each signing/notarization secret to one direct-exec consumer, removes exported secret variables before the first child process, and uses private randomized signing/notarization paths under `RUNNER_TEMP`; only non-secret signing paths cross steps through GitHub's `GITHUB_ENV` file. Decoded key material is deleted with post-delete checks, while catchable cancellation terminates the tracked notarization process group and escalates from `TERM` to bounded `KILL` before status-preserving cleanup. Mock `openssl`/`security` lifecycle tests cover success, failure, idempotent cleanup, symlink refusal, secret-environment isolation, and a `TERM`-resistant child. Strict notary parsing emits the normalized submission ID from the same verified snapshot, while JSON and text evidence replace the most-specific overlapping path first.
- A pristine empty/idle tray now uses one labelled body-level **Capture Current Settings** primary action and removes the duplicate header Capture icon; nonempty and non-idle states retain one compact header Capture action.
- Profiles Settings now keeps **New Profile** direct and moves Duplicate/Delete, Move Up/Down, and Import/**Export Saved Profiles…** into one secondary menu. Dirty drafts visibly state that Export uses the persisted document and excludes unsaved changes; injected storage coverage proves the draft is never merged into JSON.
- Setting inclusion now reads **Apply with profile**, shows Included/Not included text and a non-color symbol, and exposes setting-specific accessibility label/value/help.
- Advanced Diagnostics now uses a 520×360 minimum and 640×460 ideal contained by the 680×480 Settings minimum. About, safety, result, error, and diagnostics surfaces adapt or scroll above stable safe actions for long Korean accessibility text.
- Every Settings command routes synchronously to Profiles through one stale-safe transition. Generic profile-storage failures now use a prominent heading/icon/card with exactly one context-valid Retry Loading or Dismiss Error action, temporarily disable the profile workspace, and clear stale editor feedback, while editor-owned errors avoid duplicate global presentation.
- Routine UI noise is removed while safety state remains: duplicated Settings copy and labels, steady-state success text, low-risk visual tags, and zero-valued result counters are hidden; errors, omissions, verification failures, rollback outcomes, and accessibility metadata remain. A previously included Audio, ColorSync, or service-IPv4 value that later becomes unavailable keeps a warning and Include-off repair control.
- Normal and available-items preparation both reject zero-operation plans. Included Audio, ColorSync, and service-IPv4 changes that cannot be applied are typed omissions instead of silent drops, while already-matching values remain omission-free no-ops.
- Core Audio volume catalogs and editor capability resolution are target-device scoped, including profiles that switch the default device before applying volume.
- Profile persistence now permissions a same-directory staging file before atomic rename, so a pre-commit permission failure preserves the prior destination.
- The unused condition-editor app source and obsolete private helpers are removed; the Core condition schema, persistence, imports, evaluator, and round-trip compatibility tests remain.
- Execution-equivalence checks now canonicalize JSON object ordering while preserving exact value and rollback comparisons. Core Audio also emits sorted-key operation and rollback JSON, preventing semantically identical preparations from looping back to refreshed review.
- The opt-in adjacent live-preparation test now covers normal and available-items modes and reports only sanitized booleans on failure instead of rendering preparation payloads.
- AppKit now retains ownership of the attached `NSPopover` wrapper's nonzero origin; application synchronization maps only the hosted SwiftUI child to the wrapper's local bounds. A genuine attached-native regression protects that boundary. The public parent also preserves top/bottom safe area while filtering left/right before SwiftUI's first proposal, and the tray root owns the only 16-point content inset.
- Profile-row actions are labelled Review/Review Available because they open a preview. The preview explicitly says when no setting has changed, the separate Apply Profile button owns mutation, changed execution preflight returns to a refreshed review without mutation, and rejected confirmations show a visible reason.
- Tray first attachment now has explicit before-show/show-return/content-window/first-layout/key/final-sync stages, fixed 260/260/316/560-point height tiers, symmetric root padding, stale-generation rejection, and no arbitrary delay.
- The status-item button no longer changes width beneath an open popover: cached readiness survives background refresh, duplicate presentations are ignored, and the latest title/width is applied after close. Empty idle content is non-scrolling, Capture is icon-only with accessibility copy, every open gets a top-default scroll identity, and focus scrolling is no longer animated.
- Profile activation and group/option disclosure were removed from the product surface. Legacy false activation migrates to active applicability; Display, Audio, and Network are flat and always expanded, with unsupported controls absent rather than disabled.
- Hidden description, conditions, Input, display origin/rotation/active, audio system-output/mute, Wi‑Fi power/SSID, global IPv4, DNS, and proxy values remain round-trip compatible but are normalized dormant for new apply payloads.
- Core Audio plans default-device changes before target-device volume. Non-settable volume rows stay absent from editing, but an included changed value now produces an explicit omission instead of a silent success. Network applies last and is restored first during protected rollback.
- Production no longer uses `MenuBarExtra(.window)`, measured content height, SwiftUI root sheets/alerts, or view-lifetime tray tasks. The new popover chooses a viewport once per open session and keeps later state changes inside one scroll region.
- Settings and workflow handoffs close the tray only after the destination reports visible and key. Presentation has a bounded non-key liveness deadline and independently cancellable coalesced callers; red-close detaches before cancelling so a same-turn reopen is fresh, and stale completion cannot hide it.
- The tray top anchor is now the first content block itself, and its generation-tagged scroll request is emitted after `NSPopover.show`; reopening no longer inserts a spacing-only row or loses the first attach request.
- Profile editing keeps an app-lifetime saved/draft distinction, protects replacement and ordinary quit paths with save/discard/cancel, preserves dirty edits across external refreshes, and merges current non-editable metadata on save. The ambiguous “Update Draft from Current Settings” action and explanatory copy were removed; tray Capture remains the explicit way to create a profile from current state.
- Persistent destination windows remain visible across application/Space switching and participate in the normal window cycle. Presentation failure/cancellation releases that activation ownership, while red-close preserves the controller and draft for later reopening.
- The editor now keeps save/revert status outside scrolling content, supports `⌘S`, summarizes included values instead of option counts, hides technical identifiers behind disclosures, and uses a curated icon picker without rewriting imported symbols.
- The profile editor now uses wider, individually padded cards instead of a dense `Form`; the fixed sidebar is narrower and expanded option rows retain readable English/Korean titles at standard, minimum, and simulated large-text sizes.
- Setting-group/option inclusion remains independent of the always-expanded editor. Typed pickers and semantic sliders replace avoidable raw input; display-mode choices come from a typed ephemeral read-only catalog and are not added to profile JSON. Unsupported rotation/active-state and administrative-network values remain read-only snapshot data. Excluding a group or supported option still preserves its configured value.
- The menu keeps compact Capture and icon-only Settings/Quit controls in the top-right header and removes per-profile ellipsis, separate availability review, and manual refresh. The one primary row action is state-aware: complete plans open normal Apply and partial plans with executable operations open Apply Available, each through explicit preview.
- Apply checks app-lifetime dirty drafts before planning. Save-and-apply, discard-and-apply, and cancel preserve the existing draft contract and never silently execute a stale stored profile.
- Apply previews use friendly value descriptions and progressive technical details with adaptive sizing. Results are summarized per profile and item, followed by read-only replanning so an operation that remains necessary—or cannot be read back reliably—is not finalized as applied. Rollback reconciliation uses operation UUIDs so a rollback replaces only its matching success, and a successfully rolled-back failure remains an apply failure.
- Display rotation/active state and administrative IPv4/DNS/proxy values are preserved but normalized to excluded snapshot-only leaves at capture, local-load, and import boundaries. Empty unsupported-only groups are excluded without deleting captured values.
- Condition editing remains absent. Existing/imported conditions are preserved for schema round trips but are dormant compatibility data and do not block current manual readiness, preview, or application; automatic condition switching is not added.
- Capture now treats the Capture button as the explicit permission-request boundary for a Wi-Fi name: an undetermined Location state gets a privacy explanation and macOS prompt, while denied/restricted states offer macOS System Settings or an explicit Wi-Fi-free capture. User-facing results report applicable settings and actionable permission gaps only; snapshot-only, unreadable, and unsupported evidence no longer creates noisy partial-capture errors. The always-unsupported network service-order query was removed.
- Draft save validation reports field-specific required/range/format errors before persistence and associates localized non-color/accessibility error metadata with the affected control.
- Profiles/System/About replace the former four-tab Settings navigation. System keeps login and required permission guidance visible; diagnostics move to an on-demand advanced troubleshooting sheet with text severity. About retains source, issues, license, and privacy links; display safety retains default/Escape shortcuts and a remaining-time accessibility value.
- Profile deletion confirmation expands inside its tray profile card instead of presenting a transient system dialog. The fixed open-session viewport and single scroll region keep both confirmation actions reachable; Cancel and `Esc` clear only the pending confirmation.
- Settings now uses an app-owned resizable macOS window instead of SwiftUI's fixed-size Settings scene. The profile editor keeps one horizontal sidebar/editor anatomy from the 680×480 minimum upward, with fixed footer/action bars, at-most-one refresh adoption per presentation, hidden-window/profile scroll reset, reachable validation feedback, stable invalid-field accessibility metadata, and duplicate publication suppression.
- Apply previews reset to the top for every new or refreshed request; notices and changes share one scroll region above reachable measured actions, including at the 520×360 Korean accessibility-text minimum. Workflow headers, operations, values, results, permission, dirty-draft, and protected-change actions adapt between grids/rows and vertical stacks; the countdown remains one stable accessibility value.
- Current English/Korean deletion, capture-feedback, save-feedback, and minimum-layout screenshots plus read-only AX logs passed visual, structural, localization, and privacy review. A scoped installed interaction passed deletion Esc/Cancel/Confirm and 980→680→980 state preservation with full store rollback.
